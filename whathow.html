The 13 commits

Primitives

One recipe, five screens

The HUD, the build bar, the tower card, the Command Core panel, the profile panel, the win/lose banner — six different surfaces in css/style.css, all built from one glass recipe and one accent color instead of six bespoke looks.

Shared glass recipe
blur(18px)
backdrop-filter blur(18px) · the same two-stop dark gradient · 1px rgba(255,255,255,.06) border · the same inset+drop shadow · 18px radius. One selector list in style.css draws all six surfaces.

One accent color

#7ED9FF
HUD stat values · the docs.html HUD link · tower/room panel titles · every hover border on a build/tech/skill button · the achievement wall's "special" tier · the restart button. Ten-plus call sites, one hex value.

One button-state pattern

IdleHover DisabledMaxed
Hover: border + tint to the accent. Disabled: opacity .35–.4, cursor not-allowed. Maxed: green border, no further spend. Every interactive button in the Core screen, the profile panel and the build bar reads this way.
Found while building this page: .profile-panel (added in Phase 4) had drifted from the pattern — it duplicated the shared glass recipe line-for-line instead of joining the shared selector like every other surface. Folded it in and deleted the duplicate block. That's what this page is actually for: not just claiming consistency, but somewhere to catch it when it slips.
