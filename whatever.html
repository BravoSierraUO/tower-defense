How "active hours" is computed, honestly: commits within · hours of each other count as the same work session; a bigger gap means time off, not counted. Each session's length is its own first-to-last-commit span (floored at 15min so a single quick commit isn't ~0h). This is a disclosed estimate from real timestamps (scripts/gen-stats.mjs), not a precise time-tracker reading — there isn't one for an agent-paired workflow with dense commit bursts. A raw first-to-last-commit span (which is what this number used to be, hand-typed and left stale) would count every overnight gap as "hours" too — this doesn't.

Primitives

One recipe, five screens

The HUD, the radial build menu, the tower card, the Command Core panel, the profile panel, the About/Analytics panel, the win/lose banner, the settings panel, the avatar menu, the confirm modal, the tutorial mission banner, the wave menu — twelve different surfaces in css/style.css, all built from one glass recipe and one accent color instead of seven bespoke looks. This page is built from the same recipe — the cards below are the actual CSS values, not a mockup.

Shared glass recipe
blur(18px)
backdrop-filter blur(18px) · the same two-stop dark gradient · 1px rgba(255,255,255,.06) border · the same inset+drop shadow · 18px radius. One selector list in style.css draws all eleven surfaces — and every card on this page.

One accent color

#7ED9FF
HUD stat values · the in-game HUD's DOCS/ABOUT links · tower/room panel titles · every hover border on a build/tech/skill button · the achievement wall's "special" tier · the restart button · the About panel's category bars (four opacity steps of this one hex, not four new hues) — and this page's own accent, replacing an earlier draft that used an unrelated pink. A dozen-plus call sites, one hex value.

One button-state pattern

IdleHover DisabledMaxed
Hover: border + tint to the accent. Disabled: opacity .35–.4, cursor not-allowed. Maxed: green border, no further spend. Every interactive button in the Core screen, the profile panel and the build bar reads this way.

One panel-close pattern

.panel-close-btn (a top-right ✕, reusing the mission banner's round-icon look) and .panel-footer-close-btn (reusing the settings-button look) bracket a .panel-scroll middle — so the profile, About, and Settings panels each get two fixed exits regardless of scroll position, from two existing button looks instead of a new one.
Found while building the previous version of this page: .profile-panel (added in Phase 4) had drifted from the pattern — it duplicated the shared glass recipe line-for-line instead of joining the shared selector like every other surface. Folded it in and deleted the duplicate block. Then this page itself drifted from the pattern the same way — it invented its own pink/rounded-font palette instead of the game's real one. Re-themed here to match. That's what this section is actually for: not just claiming consistency, but somewhere to catch it when it slips, including in the doc page auditing it.
Found during the v2.8 code-quality pass: the profile, About, and Settings panels had no close affordance at all — only the same hotkey (P/O/S) that opened them, which isn't discoverable, so a player could get stuck looking at one. Fixed by adding the pattern above as a shared primitive rather than patching the one panel that got reported.
